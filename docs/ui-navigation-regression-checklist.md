# 导航 UI 回归检查清单

用于每次发布前快速验证「现代布局 + 经典布局」导航区域是否正常。

## 基础检查

- 品牌 Logo 图标与文案显示正常（顶部与移动端）。
- 深色/浅色切换按钮可点击，图标状态正确。
- 切换布局按钮可点击，现代/经典布局可来回切换。
- 退出登录按钮可点击，能正确回到未登录状态。

## 现代布局（NavBar）

- 桌面浮动导航：当前路由高亮胶囊显示正常。
- 桌面导航链接 hover/active 文案颜色正确。
- 右侧操作按钮圆角/hover/focus ring 一致。
- 移动顶部操作区按钮样式与桌面风格一致。
- 移动底部导航 active 状态与图标缩放正常。

## 经典布局（Header）

- 顶部品牌区域与现代布局视觉一致。
- 已登录：显示设置、切换布局、退出登录。
- 未登录：显示 GitHub 按钮和登录入口（自定义登录路径场景除外）。
- iOS 安全区顶部 padding 正常，无状态栏遮挡。

## 交互与可访问性

- 所有图标按钮均有可读 `aria-label`。
- 键盘可聚焦按钮并看到焦点样式（focus ring）。
- 减少动态效果设置（`prefers-reduced-motion`）下无突兀动画。

## 本地调试专项

- `npx wrangler pages dev dist --port 8999` 启动后首页不白屏。
- 控制台无 `undefined is not a non-null object` 等初始化错误。
- 强刷后页面仍可正常加载（避免旧 SW 缓存影响）。
